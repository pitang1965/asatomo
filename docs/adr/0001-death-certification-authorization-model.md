# ADR-0001: 最後のメッセージ開示の権限モデル（死亡認定）

- ステータス: Accepted
- 日付: 2026-07-13

## 背景（Context）

アサトモの差別化機能は、[[本人]]の死後に[[最後のメッセージ]]を[[受取人]]へ開示することである。既存のデッドマンズスイッチ（純粋タイムアウト型）と異なり、**人間の判断を介在させて誤爆を防ぐ**ことを核とする。

この設計には相反する複数の力が働く:

- **中心的トレードオフ**: 誤爆防止を強めるほど（人数・合意要件を上げる）、本物の死で開示が起きにくくなる／デッドロックする。開示を確実にするほど、誤爆・いたずらのリスクが上がる。
- **決定的な非対称**: 本人が真に死んでいれば本人はキャンセルできない（携帯も切れる）。「本人ワンタップ取消」は*偽陽性にしか効かない*。真の死では承認者（見守り者）が最後の砦になる。
- **利益相反**: 開示で利益を受ける[[受取人]]に死亡確認をさせると、「受け取りたさに開錠する」経路が生まれる。相続法の「執行者 ≠ 受遺者」と同じ問題。
- **単独承認問題**: 見守り者が1人だと、その1人の判断だけで開示が成立し、「一人の誤操作・いたずらで発動しない」という要件を原理的に満たせない。
- **安全側の向き**: 偽の「生きている」は偽の「異常かも」より危険（本アプリの存在理由は"本物の死"を取りこぼさないこと）。

## 決定（Decision）

**能力の分離（非対称）**

- 「見守る」（生存監視＋死亡認定への投票）は*人*に紐づく役割 = [[見守り者]]。
- 「受け取る」は*人*の固定属性ではなく、メッセージ宛先指定から派生する動的な状態 = [[受取人]]。
- **投票権は見守り者にのみ紐づく。** 見守り者は受取人を兼ねてよい。純粋な受取人（見守り者でない）は投票できず、「連絡が取れない」等の**懸念フラグ**を上げてエスカレーションを起動できるのみ。

**開示の成立条件**

- [[最後のメッセージ]]の開示機能は、**独立した見守り者2人以上**が有効な本人でのみ有効化する。0〜1人の本人は生存確認は使えるが開示はロックし、UIで不足人数を明示する。
- **見守り者ゼロの本人（純粋タイムアウト型）は認めない。** 承認者不在は本アプリの「人間の判断で誤爆を防ぐ」思想の放棄に等しい。
- クォーラム = **2人以上の賛成、かつ生存している見守り者の過半数**。全員一致にはしない。
- 一定期間ログイン/応答の無い見守り者は「休眠」として**定足数の分母から自動除外**する。除外は全見守り者に可視化する。

**四重防御**

1. 独立見守り者2人以上、かつ 2. 過半数合意、かつ 3. 猶予期間（24〜72時間・本人に通知）、かつ 4. 本人のワンタップ取消。
偽陽性は 3・4 で、真の死は 1・2 で担保する。

**安全原則**: システムは「沈黙」ではなく「エスカレーション」の側へ倒す。代理確認（見守り者による生存の証言）は常に監査可能・全見守り者に可視化し、静かに警報を消せないようにする。

**法的位置づけ**: 開示物は「最後のメッセージ」であり法的な遺言ではない。口座番号・PIN等の守るべき秘密は書かないよう警告する。

## 状態機械の不変条件（派生決定）

死亡認定の状態遷移（正常 → 未応答 → 見守り者アラート → 投票中 → 認定成立(猶予) → 開示）は、次の不変条件に従う:

- **A. 本人の[[生存シグナル]]は、開示前ならいつでも全てを覆す。** 本人自身が生の直接証拠（アラーム停止・ごはん・ログイン等）を発したら、投票中でも猶予中でも即座に死亡プロセスを中断し「正常」へ戻す。
  - **(2026-07-18 精緻化) 覆しの基準は発生時刻（occurredAt）であって到着時刻ではない。** シグナルは発生時点の生存証拠であり、オフライン再送などで遅延到着した古いシグナルは lastSignalAt を発生時刻まで前進させ段階を再評価するだけで、自動キャンセルはしない。覆しが発動するのは、発生時刻基準の経過時間が検知窓内に戻る場合のみ。理由: 「端末の稼働 ≠ 本人の生存」（例: 死亡後に第三者が端末を充電・接続し、キューが発火するケース）。偽の「生きている」は偽の「異常かも」より危険、という本ADRの安全側の向きに従う。
  - **(2026-07-23 精緻化) リアルタイムの第三者シグナルは受容する残存リスクとする。** occurredAt 基準の stale 判定が弾くのは遅延到着（オフラインキューの後発火）だけで、第三者が生きた端末をいま操作して出すシグナル（「元気ですか？」プッシュのタップによるアプリ起動を含む）は stale ではないため正当なシグナルとして受理し、進行中エピソードを覆す。全生存シグナル（ごはん・アラーム停止・Webチェックイン等すべて）は「**端末操作の証拠であって本人性の証拠ではない**」性質を共有し、操作可能な端末を握る者なら誰でも任意の種別を出せる。ここを塞ぐには全シグナルにシグナルごとの生体認証を課すしかなく、「ただの目覚まし時計」UX（本人側の監視感の払拭）を壊すため**採らない**。プッシュタップだけを塞いでも第三者は次に「ごはん」ボタンを押せるので穴は塞がらない。悪意ある同居人が能動的に死を隠すケースは、段階2で見守り者が実際に様子を見に行く層が拾う。**砦はシグナルの偽造耐性ではなく見守り者の人間判断**、という本ADRの中核方針の帰結。
- **B. [[代理確認]]は"覆さ"ないが"知らせる"。** 投票中の見守り者による生存の証言は、一人の証言で他の投票を黙って打ち消さない。全投票者に可視化し、各自が票を取り下げられる。自動中断するのは A（本人の直接シグナル）だけ。
- **C. 開示は不可逆の終点。** 受取人が復号済みになるため取り消せない。**猶予期間が引き返せる最後の地点**であり、UIでも重く扱う。
- **D. 見守り者が2人未満に減ったら開示を自動ロックし本人に通知。** 辞退・休眠で有効な見守り者が2人未満になったら開示機能をロックし、本人に不足を知らせる。

## エスカレーションのタイミング（既定値・派生決定）

方針: **早い段階は安く（本人へ軽い再通知）、社会的コストの高い段階の"手前"に緩衝を置く**。数値は調整可能な既定であり固定契約ではないが、**安全の床は本人でも外せない**。

| 段階 | 既定 | 可変幅 | 調整権 |
|---|---|---|---|
| 最後のシグナル → 段階1（未応答・本人再通知） | 30h | 18〜48h | 本人 |
| 段階1 → 段階2（見守り者アラート） | 12h | 6〜24h | 既定固定寄り |
| 段階2 → 投票成立の"床" | アラートから最低12h | — | 固定（安全の床） |
| 認定成立 → 開示（[[猶予期間]]） | 48h | 24〜72h | 本人 |

- 判定は[[生存シグナル]]の**経過時間ベース**（時刻ではない）なので、夜勤・不規則な生活リズムを自然に吸収する。極端に不規則な場合は判定窓を手動で広げる。
- 見守り者アラートは最短でも約42h後 = *人間が実際に様子を見に行く*生存確認の中核の瞬間。開示は最短でも約4日後（緊急救助アプリではないので目的に適う）。

**将来余地（v2）**: 「変則期間モード」（旅行モードの軽い親戚。「今週は勤務が変則」と宣言して一時的に判定窓を広げ、見守り者に可視化）。MVPには入れない。自動学習による判定窓の適応もv2候補。

## 結果（Consequences）

**正**

- 「一人が自分宛の遺言を単独開錠する」経路が構造的に塞がる。
- 純粋タイムアウト型に堕さず、差別化の核（人間の判断）を保てる。
- 休眠除外により、見守り者1人の音信不通で開示が永久デッドロックする事故を避けられる。

**負 / 要フォロー**

- 「頼れる人が1人しかいない孤独な単身者」は、2人目を作るまで開示機能を使えない。→ ロックは罰ではなく安全な開示の最低条件と位置づけ、**相互見守り・なふだ経由・アプリ内招待で2人目を作る導線**を主機能としてUX化する必要がある。
- 「休眠」判定・定足数の動的計算・可視化にロジックコストがかかる。
- 過半数（非全員一致）ゆえ、悪意ある見守り者が過半数を占める極端ケースは残存リスク。猶予期間＋本人取消＋可視化で緩和する。

## 検討した代替案（Alternatives considered）

- **受取人に死亡確認をさせる**: 利益相反。却下。受取人には懸念フラグのみ許す。
- **全員一致クォーラム**: 1人の音信不通でデッドロック。却下。
- **見守り者1人でも開示可**: 単独開錠を原理的に防げない。却下。
- **見守り者ゼロを純粋タイムアウトで許容**: 差別化元のデッドマンズスイッチそのものに退行。却下。
