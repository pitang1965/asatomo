# CONTEXT — アサトモ 用語集

このファイルはアサトモの**ドメイン言語（ユビキタス言語）**を定義する用語集です。
実装の詳細（テーブル設計・API・技術選定）は書きません。それらは `docs/adr/` に記録します。

---

## 本人（Subject）

見守り**関係**において「見守られる側」の役割。生存シグナルを発する主体。

- 役割はアカウント種別ではなく**関係上の役割**である。同一ユーザーが、あるつながりでは本人、別のつながりでは見守り者になりうる。
- **UI での言い換え**: [[見守り者]]に見せる文脈では、総称が要る所（一覧ヘッダー等）は「**見守っている人**」、個別に指せる所（カード・詳細）は**実名**を最優先とする。能動態で視点を固定し、「相手／先」等の物扱いの語感は避ける。「本人」は内部の正式名称であり UI 表面には出さない。
- **混在文脈での主語明示**: 本人向けの画面（「見守ってくれる人」が近くに現れる文脈）に見守りセクションを置く場合は、向きの混同を防ぐため「**あなたが見守っている人**」と主語を付ける。一字違いの「見守ってくれる人／見守っている人」を同一画面で無印のまま並べない。
- 関連: [[見守り者]] / [[つながり]] / [[生存シグナル]]

## 見守り者（Watcher）

「見守る役割」を持つ[[つながり]]。本人の状態を閲覧し、異常時に連絡を取り、[[死亡認定]]の**投票権を持つ**。

- 役割はアカウント種別ではなく、つながりの上の役割。
- **[[受取人]]を兼ねてよい**（原則、[[最後のメッセージ]]を受け取る）。開示の独立性は「集合を分ける」ことではなく**クォーラム（見守り者2人以上の合意）**で担保する。
- **UI での言い換え**: [[本人]]に見せる文脈では能動態で「**見守ってくれる人**」と表す（「あなたを見る側」を明示）。「大切な人」は役割を特定しない**詩的コピー専用**であり、[[本人]]・見守り者いずれの正規の言い換えにも使わない（見せる相手の視点により指す役割が反転し、厳密に分けた両概念を UI で再び混同させるため）。
- 関連: [[本人]] / [[つながり]] / [[受取人]] / [[死亡認定]]

## つながり（Connection）

[[本人]]が自分のアサトモのサークルに加えた相手。本アプリ文脈での「つながり」であり、なふだ（名刺アプリ）の「つながり」とは**別文脈の別概念**（DB分離・API連携）。

- **有向**: 「AがBをつながりに加える」と「BがAを加える」は独立。相互見守りは2本のつながりで表現。一方向（非対称）でも双方向（相互）でも表せる。
- 各つながりは「見守る役割を持つか（→[[見守り者]]）」「[[最後のメッセージ]]の宛先か（→[[受取人]]・派生）」という性質を持つ。**信頼の階層が混在**するため、umbrella としてあえて中立的で軽い語を用いる。
- なふだのつながりからアサトモへ招く導線は、名詞の新設ではなく**明示的な承諾を伴う「昇格」**として扱う（軽い名刺関係を黙って生死監視へ引き上げない）。
- 関連: [[本人]] / [[見守り者]] / [[受取人]] / [[相互見守り]]

## 相互見守り（Mutual Watching）

2人が互いに相手の[[見守り者]]になる状態（[[つながり]]を2本、双方向に張る）。

- アサトモの「友」のアイデンティティであり、**標準のオンボーディング経路**。似た境遇の単身者どうしがゆるく見守り合うことで、開示に必要な見守り者2人を自然に満たす（[ADR-0004](docs/adr/0004-product-layering-daily-first.md)）。
- 関連: [[つながり]] / [[見守り者]]

## 招待（Invitation）

見守り関係へ相手を誘う、**まだ相手が確定していない開いた申し出**。[[つながり]]とは別概念（つながりは相手ユーザーが確定して初めて成立するエッジ）。

- リンクで送れ、受け取った相手は**asatomo未登録でもよい**。相手がログインして初めて相手が確定する（アプリ不要の標準経路。[ADR-0004](docs/adr/0004-product-layering-daily-first.md)）。
- 申し出には寿命があり、**承諾**されると[[つながり]]へ**昇格**する（[[相互見守り]]なら双方向の2本）。**辞退・取消・失効**で閉じる。
- なふだからの昇格も、この招待の承諾（明示同意）を通す（[ADR-0003](docs/adr/0003-nafuda-integration-boundary.md)）。軽い名刺つながりを黙って生死監視へ引き上げない。
- 関連: [[つながり]] / [[相互見守り]] / [[見守り者]]

## 見守り解除（Watch Revocation）

承諾済み（accepted）の[[見守り者]]エッジ1本を終える遷移。**有向エッジ1本に対する操作**であり、関係全体の終了ではない（[[相互見守り]]は2本なので、片方だけ解けて非対称になりうる）。

- 同一の遷移を**2つの端から**引ける。結果状態（見守り無効化＋[[死亡認定]]の開示可否の再計算＝不変条件D）は同じで、違うのは**起点と、通知の向き**。結果は同じでも通知は**対称ではない**（下記の安全原則）。
  - **本人端**（本人がその見守り者を外す）— UIの言い換え（本人視点。相手は「見守ってくれる人」）:「**見守りをお願いするのをやめる**」。
  - **見守り者端**（見守り者が自分から降りる）— UIの言い換え（見守り者視点。相手は「見守っている人」）:「**見守りをやめる**」。
- 「外す／解除する」は相手を物扱いする語感が出るため UI 表面には出さない。「見守り解除」は**内部の正式名称**としてのみ用いる。
- 解除は安全網を減らす行為。通知は**非対称**（起点＝当事者はすでに知っているので、通知はもう一方へ向く）:
  - **見守り者端**で解除され本人の網が黙って縮む場合は**本人へ通知**する（誰が抜けたかを沈黙させない）。
  - **本人端**では、外された見守り者へは**通知しない**（本人が起点＝当事者で承知の上。相手の可視性から黙って消える）。健全な整理をためらわせないため。ただし解除で本人自身の網が開示ライン（[[見守り者]]2人＝不変条件D）を割る場合は、その結果を**本人に**その場で示す（沈黙より通知は本人自身にも向く）。
- [[招待]]の辞退（承諾前）や pending の取消とは**別**。見守り解除は**承諾後**を指す。
- 関連: [[見守り者]] / [[つながり]] / [[相互見守り]] / [[死亡認定]] / [[アカウント削除]]

## アカウント削除（Account Deletion）

あるユーザーが、[[本人]]・[[見守り者]]・[[受取人]]のどの役割も含めてアサトモから**完全に退場**すること。[[見守り解除]]（エッジ1本を終える）を**内包するが、それより大きい別概念**。

- **即時・不可逆。** 猶予付き復元（ソフト削除）は作らない。安全網グラフに「半分消えた」曖昧な状態を残さず、即時にクリーンに畳む。誤削除への防御は可逆性ではなく**情報つきの摩擦**——削除前に依存者ごとの結果を見せる確認——で与える（[ADR-0007](docs/adr/0007-account-deletion-model.md)）。
- あなたが[[見守り者]]を務める全エッジの**一括[[見守り解除]]（見守り者端）**でもある。見守り解除の**非対称通知**をそのまま適用: あなたが抜けて網が縮む[[本人]]へ「〇〇さんが見守りをやめました」と通知する。開示ライン（[[見守り者]]2人＝不変条件D）を割る本人には、その結果を確認画面で示す。
- あなたを見守っている[[見守り者]]には「〇〇さんはアサトモの利用をやめました」と、[[死亡認定]]/開示とは**別物の穏当な**通知を出す（突然の消失を緊急事態と誤読させない）。
- あなたの[[最後のメッセージ]]・[[受取人]]指定は**破棄**する（開示は永久に起きない）。[[受取人]]には通知しない（宛先の存在を告げること自体が故人の意図の漏洩になり、渡せる中身もない）。
- 過去の（開示済み含む）[[死亡認定]]に残るあなたの投票・[[代理確認]]も**完全消去**する（プライバシー優先。アプリ内に監査の消費者が存在しない）。
- **UI 表面でも率直に「アカウントを削除する」と表す（例外）。** 「見守り解除」で用いた柔らかい言い換え規則はここには適用しない——削除だと明確に認識できることが安全・信頼・ストア要件だから。思いやりは確認画面の警告文が担う。
- 「見守られるのを一時的にやめたい」（→[[旅行モード]]・監視の一時停止）とは**別**。完全退場のみを指す。
- 関連: [[見守り解除]] / [[本人]] / [[見守り者]] / [[受取人]] / [[最後のメッセージ]] / [[死亡認定]]

## 生存シグナル（Liveness Signal）

**本人自身**が発する、生存の**直接証拠**となるイベント。

- 種別: アラーム停止 / 「ごはん」/「おやすみ」/「いってきます」（外出）/「ただいま」（帰宅）/ アプリ起動 / Webチェックイン / （任意）端末ロック解除。
- アプリ起動は自動で記録される。透明性の原則に従い、自動記録があることを本人に明示する（隠れて記録しない）。
- 種類を問わず「本人が最後に生存を示した時刻」を更新する。異常判定は**時刻（何時か）ではなく、そこからの経過時間**で行う。
- シグナルは**発生時点の証拠であり、到着時点の証拠ではない**。遅れて届いたシグナル（オフライン再送等）は「最後に生存を示した時刻」を発生時刻まで前進させるだけで、進行中の[[死亡認定]]を無条件には覆さない。覆るのは、発生時刻基準の経過時間が異常判定の閾値内に戻る場合のみ（[ADR-0001](docs/adr/0001-death-certification-authorization-model.md)）。
- この「経過時間ベース」という性質により、**夜勤・不規則な生活リズムを自然に吸収**する（何時に活動しても、活動しさえすれば時計はリセットされる）。
- [[代理確認]]（見守り者による間接証拠）とは**性質が異なる別概念**。混同しない。
- 関連: [[本人]] / [[代理確認]]

## 代理確認（Watcher Attestation）

**見守り者**が「本人が生きているのを確認した」と申告する**間接証拠**。[[生存シグナル]]とは区別する。

- 位置づけ: 原則としてエスカレーション（見守り者への「連絡してみて」通知）への**解決アクション**。日常の汎用シグナル源にはしない。
- 例外: [[完全受動型の本人]]では、代理確認が唯一の生存確認源になりうる。
- 常に「誰が・いつ確認したか」を記録し全見守り者に可視化する（怠慢・偽証言を沈黙させないため）。任意で根拠メモを残せる。
- 関連: [[見守り者]] / [[完全受動型の本人]]

## 完全受動型の本人（Fully-Passive Subject）

アプリもWebチェックインも使わず、生存の確認が[[代理確認]]のみに依存する[[本人]]。

- 弱い見守りモードであることを**本人が承知の上でオプトイン**する。
- 関連: [[代理確認]]

## 最後のメッセージ（Final Message）

[[本人]]が生前に用意し、[[死亡認定]]の開示成立後に[[受取人]]へ届く、想い・感謝・別れの言葉。

- **「遺言」ではない。** 法的な財産処分の効力は持たない（アプリ内に明示）。口座番号・PIN等の守るべき秘密は書かないよう警告する。
- 開示条件が成立するまで運営者も読めない暗号化保存とする。本人は生前いつでも編集・削除できる。
- 関連: [[受取人]] / [[死亡認定]]

## 受取人（Recipient）

**あるメッセージの宛先として本人が現在選んでいる人**。[[最後のメッセージ]]の宛先指定から**派生する動的な状態**であり、人に固定的に貼られる役割ではない。

- 本人はいつでも宛先を変更でき、受取人は増減する（「今は送る予定 → 送らない予定」への変更も自然）。
- 見守りをする人（[[見守り者]]兼任）と、しない人（純粋な受取人）がいる。
- **純粋な受取人（見守り者でない）は[[死亡認定]]の投票権を持たない。** 「連絡が取れない」等の懸念フラグを上げることはできる（見守り者のエスカレーションを起動する）。
- 「見守る」（人に紐づく役割）と「受け取る」（メッセージ宛先から派生）は**非対称**。混同しない。
- 関連: [[見守り者]] / [[最後のメッセージ]] / [[つながり]]

## 死亡認定（Death Certification）

[[見守り者]]が「本人が亡くなった」と人間の判断で認め、[[最後のメッセージ]]の開示に至るプロセス。

- 純粋なタイムアウト型デッドマンズスイッチと異なり、**人間の判断を介して誤爆を防ぐ**のが本アプリの核。
- 誤爆防止のため多段防御を持つ（最低2人の独立見守り者・過半数合意・猶予期間・本人取消）。**成立条件の具体値は [ADR-0001](docs/adr/0001-death-certification-authorization-model.md) を参照**。
- 用語は重いので、UIの文言は慎重・段階的にする（内部の正式名称としてのみ用いる）。
- 関連: [[見守り者]] / [[受取人]] / [[最後のメッセージ]]

## 合言葉（Passphrase）

本人が[[受取人]]ごとに設定する、[[最後のメッセージ]]を復号するための秘密。**本人と受取人のふたりが共有する1対のもの**（山→川）。

- 運営者は合言葉を保持せず、原理的に平文を読めない（ゼロ知識）。開示後に受取人が入力して端末側で復号する。
- 「その受取人だけが知っていること」を合言葉にすると、生前の事前共有が不要になる。ヒント（質問）には答えの形式をダミー例で添える（例:「最初に飼った鳥の種類と名前は？ 例：カラスのガーちゃん」）と、受取人が表記に迷わない。
- **合言葉を失うとメッセージは永久に復元できない**（運営者も助けられない）。仕組みの詳細は [ADR-0002](docs/adr/0002-final-message-encryption-key-custody.md)。
- 誰とも共有しない本人専用の鍵は合言葉とは呼ばず、[[編集用パスワード]]と呼ぶ（1対でないため）。
- 関連: [[最後のメッセージ]] / [[受取人]] / [[編集用パスワード]]

## 編集用パスワード（Author Password）

本人だけが知る、[[最後のメッセージ]]を生前に読み返し・編集・保存するための鍵。全メッセージ共通で1つ。

- [[合言葉]]（受取人と共有する1対の秘密）とは別物。誰にも教えない。UI でも「合言葉」とは呼び分ける。
- ゼロ知識のため、保存した本文は本人であってもこのパスワードなしでは読めない（サーバは平文を返せない）。
- 忘れた場合、既存メッセージの読み返し・編集は永久にできなくなる（新規作成は可能）。
- 技術的には DEK を本人鍵でもラップする authorWrappedDek に対応（[ADR-0002](docs/adr/0002-final-message-encryption-key-custody.md)）。
- 関連: [[合言葉]] / [[最後のメッセージ]]

## 旅行モード（Travel Mode）

本人が計画的な不在・生活リズムの変化の間、[[死亡認定]]へ向かう自動エスカレーションを一時的に抑制する状態。

- 誤報防止のための必須機能。ただし「沈黙」を作るため、安全原則（沈黙より通知）に従わせる: **期限付き・自動で監視再開・全見守り者に可視化**する。これにより沈黙が「いつまで待てばよいか」という情報に変わる。
- 関連: [[生存シグナル]] / [[死亡認定]]

## 近況（Recent Activity）

[[見守り者]]の画面に表示される、本人の最近の生存の様子。**過去形＋相対的な経過時間**で表す（例:「約2時間前に食事をしました」）。

- リアルタイムの「今〇〇中」表示はしない。過去形・経過時間の枠組みが「今この瞬間を覗かれている」監視感を消し、「ゆるく知ってる安心」だけを残す。絶対時刻（14:03等）は出さない。
- **留守を推測させるシグナル（「いってきます」等）は、近況ではぼかした汎用表現で見せる**（例:「元気に過ごしています」）。記録には真の種別を残すが、見守り者への表示で「家が無人」を直接開示しない。記録と表示は分離してよい。
- 本人には「この行動がつながりに伝わる」ことを透明にする（隠れて記録しない）。「監視感の払拭」は本人側UX（ただの目覚まし時計に見える）で担保する。
- 関連: [[プレゼンス]] / [[生存シグナル]] / [[見守り者]]

## プレゼンス（Presence）

「食事中・就寝中」等の一時的な状態。主に**内部の判定精度向上**に使う（例: 就寝は翌朝のアラーム停止の予告として判定に活かす）。

- 見守り者にはリアルタイムに晒さない。外部に見えるのは[[近況]]（過去形・粗い鮮度）まで。
- 関連: [[近況]] / [[生存シグナル]]

## スマホアプリ / 見守りWeb（プラットフォームの呼び分け）

2面構成: **スマホアプリ**（Android ネイティブ。目覚まし・[[生存シグナル]]等、[[本人]]の日常の道具）と
**見守りWeb**（Web アプリ。[[見守り者]]の完全な面＋まれで重いフローの唯一の実装）。理由は
[ADR-0006](docs/adr/0006-platform-seam-frequency.md)。

- **iPhone ユーザーは 見守りWeb だけで運用する**（スマホアプリを持たない）。ゆえに
  **全[[本人]]に届くコピー（無応答ナッジ等）で「スマホアプリを開いて」と限定しない**
  （iPhone 本人には開くアプリが無い）。「アプリか見守りWebを開いて」等、経路非依存にする。
- **Web 側を「アプリ」と呼ばない**（「見守りWeb」または「Web」）。素の「アプリ」はスマホアプリを指す。
  見守り者は Web 上でコピーを読むため、素の「アプリ」は取り違えられうる。
- 例外: アプリからのログアウトのように**構造上 Android でしか起こらない事象**は「スマホアプリ」と明示してよい。
- 「モバイルアプリ」は開発者・マーケ語。生活者コピーは「スマホアプリ」。
