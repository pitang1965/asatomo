## 本人端の[[見守り解除]]UI＝「見守りをお願いするのをやめる」（Web・今回）

- 発見: 2026-07-20（見守り者ゼロ時コピーの実機確認中、ゼロに戻す手段が無く判明）。2026-07-21 のグリルで設計確定。
- 事実: `revokeWatcher`（本人が見守り者を外す）はドメイン＋API（[POST /connections/revoke](src/api/router.ts)、[handlers.ts](src/api/handlers.ts) `revokeWatcher`）まで在るが、**UI がどこにも無い**。
- 置き場所: **Web の新規「つながり整理ページ」**（[ADR-0006](docs/adr/0006-platform-seam-frequency.md): 稀・管理的＝Web）。Web トップは「あなたが見守っている人」を列挙中。逆向きの「あなたを見守ってくれている人」を同居させない（CONTEXT の一字違い同居禁止）。トップには静かなリンクだけ置き、一覧＋やめるは別ページ。誘う導線（見守り合いに誘う）はトップのまま。
- 一覧の意味: **今わたしを見守ってくれている人（accepted のみ）**。外したら行は消える。再度頼むのは通常の招待リンクに戻る（revoked→再昇格は [connections.ts](src/domain/connections.ts) が処理済み）。
- 外す瞬間の安全（決定B）: 確認は常に出す。加えて**その1人で開示ライン（生存見守り者2人＝不変条件D）を割る場合だけ**、結果を平易に警告（「見守ってくれる人が1人になり、最後のメッセージを届けられなくなります」）。事前チェックが要る（`revokeWatcher` は外した後の可否しか返さないので、ページ側で残る生存見守り者数を1つ数える）。休眠を除く「生存」数で判定する（`countLivingWatchers`）。
- 通知: 外された見守り者へは**通知しない**（本人が当事者。CONTEXT の非対称を参照）。
- テスト実務メモ: 現状ゼロ状態を作るには DB で該当 `connections` 行の `watcher_status` を `revoked` にする（Drizzle Studio / Neon いずれも同一DB）。

## 見守り者端の[[見守り解除]]＝「見守りをやめる」（別途・設計確定 2026-07-21）

- 事実: 見守り者が「自分から降りる」方向は**どの層にも無い**。`revokeWatcher` は本人所有アクション（`subjectUserId` で照合）。見守り者端は **`otherUserId=自分` で自分のエッジを終える別経路**（新ドメイン関数、例 `leaveWatch`）。不変条件Dを再計算し、`{ subjectUserId, disclosureLocked, watcherName }` を返して通知を駆動する。
- 通知経路の前提修正: 「未配線」は誤り。[notifier.ts](src/notify/notifier.ts) の `notifySubject`（**プッシュ優先・メール fallback**）が既にあり、そのまま乗る。`notifySubjectDisclosureLocked` は定義済みだが**本番未使用**（呼び出しはテストのモックのみ）。配線は薄い。
- 置き場所（決定A）: **Web の見守りダッシュボード**（[index.tsx](src/routes/index.tsx) の `WatchDashboard`）の**行メニュー/詳細の中**。「無事です／連絡がつきません」と同階層に並べない（生死に関わる面での誤タップ回避）。稀・管理的＝Web（ADR-0006）。本人端の整理ページとは別の面（見る人が見守り者自身）。
- 本人への通知（決定B）: **降りたら毎回**通知し、**必ず降りた人を名指し**（CONTEXT「誰が抜けたか沈黙させない」）。開示ライン（生存2人）を割った時だけ文面を強める（＋「最後のメッセージを届けるにはもう1人必要」）。**新しい通知意図 `notifySubjectWatcherLeft(subject, 降りた人名, disclosureLocked)`** を1本足す（既存の匿名・招待前提の `notifySubjectDisclosureLocked` は流用しない）。
- 相互見守りの向き（決定C）: **1本だけ終える**（用語集「有向エッジ1本」）。束ねて相互解消はしない（片方向のつもりで自分の安全網を削る事故を防ぐ）。確認文で向きを明示し積極的に安心させる:「あなたが〇〇さんを見守るのをやめます。**〇〇さんがあなたを見守るのは続きます**」。相互を完全に畳むのは、別途「整理ページ」で本人端の操作を重ねる。
- 降りる人の確認画面（決定D）: 相手の**見守り人数は見せない**（他人の私的な安全体制。降りる人に数字は不要）。「**〇〇さんには、あなたが見守りをやめたことをお知らせします**」とだけ正直に示す（こそこそ抜けない／罪悪感で縛らない）。
- 復帰（決定E）: **自分では戻れない**。復帰は本人（被見守り者）が再招待→承諾の通常フロー（ADR-0005 の同意モデル。投票権者を同意なく復活させない）。降りたら相手のダッシュボードから消える（accepted のみ表示）。（任意）押し間違い対策に降りた直後だけ数十秒の取り消しを出すのは同意モデルを侵さない。
